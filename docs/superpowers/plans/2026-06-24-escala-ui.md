## Regras Firestore (Task A)
```
match /special_scales/{id} {
  allow read:  if isAuth() && hasProfModule();
  allow create, update: if isAuth() && (isAdmin() || isSuperv());
  allow delete: if false;
}
match /scale_preferences/{id} {
  allow read: if isAuth() && hasProfModule();
  // colaborador grava a SUA preferência; gestão grava qualquer uma
  allow create, update: if isAuth() && (
    isAdmin() || isSuperv() ||
    request.resource.data.personId == uData().professorId
  );
  allow delete: if false;
}
match /fairness_counter/{id} {
  allow read:  if isAuth() && hasProfModule();
  allow write: if isAuth() && (isAdmin() || isSuperv());
  allow delete: if false;
}
```
Deploy: `firebase deploy --only firestore:rules --project staging` (pedir OK ao usuário antes — ambiente de homologação). Validar via REST (padrão `scripts/validate-*.js`) ou no preview.
